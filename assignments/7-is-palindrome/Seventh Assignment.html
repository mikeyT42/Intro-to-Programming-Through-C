Introduction:

This assignment is going to be a chance to work more with arrays. Really, it will be a way to iterate through an array in 2 directions at the same time: you will walk an array backwards and forwards in the same loop.

This program will check if a single word is a palindrome. A palindrome is a word--or really a string of text, but we're keeping it to a single word--that is spelt the same backwards and forwards. A simple example is, "huh" or even, "radar": there are many words that you can Google to see and test in your program. You will accept string input from the user, and then output whether or not the string is a palindrome. Please show your output in a block comment at the bottom of your program.

Enumerations:

There will be an enumeration for this program: it will be called loop_control. This enumeration will represent the 2 states of our loop logic: one state being to CONTINUE loop execution and the other state being to STOP loop execution. These will be used in the do-while loop's condition.

The next enumeration will be to handle validation error codes. The name of this enumeration will be validation_codes. The values will be OK, to represent validation passing; WHITESPACE, to represent any whitespace character error; NUMBER, to represent when there is a digit character error.

main:

You will output a banner and a salutation. You will have an input loop function that returns CONTINUE or STOP on whether to exit the loop or not. The function it will call until STOP will be called input_loop(void). This will be inside of a loop defined in the main function.

input_loop(void):

This function is where the magic will happen. It will return a loop_control value representing whether or not to continue gathering input or not from the user: the return value controls the outer loop inside of the main function. This function will be the one to accept user input, check if the user wants to exit--our sentinel value will be '\n'--validate the input, clean the input, check if it's a palindrome, and then exit to continue gathering user input.

If the user wants to exit; do not validate, clean, or check if the input is a palindrome: simply return STOP to exit the outer loop in main. If the user entered a word; then validate, clean the word, check if it's a palindrome, and then return CONTINUE to continue executing the outer loop in main. Be sure after checking if the input is a palindrome to output the result to the user and then return.

validate(const char input[]):

This function will validate that the user has input a single, valid, word. This function will return a validation_codes type. If there is any space character (' ') in the string, then a single word was not entered; return WHITESPACE if this is the case. Also check for if the string contains a number and return NUMBER. Return OK if no error cases are met.

In the input_loop function, show the user an error message about the error that occurred: something like, "you entered non-valid input." Use a switch statement to catch the different error cases and output specific messages. If an error occurred, then return CONTINUE in the input_loop function to exit and then continue gathering input; otherwise continue forward to the next step.

clean_input(char input[]):

This function has a couple simple tasks. The first being to get rid of the newline character at the end of the string. If you don't replace this with the null-terminator character, there will be issues later on!

Also, normalize all of the characters to be uppercase or lowercase. Essentially, loop through the string after fixing the newline character and reassign each character at each index with its upper or lowercase version: just choose one.

is_palindrome(const char input[]):

This function is the heart of the program. This is where we check if the input array is a palindrome or not. If it is, return true; otherwise return false.

You will need to define a loop that iterates over the input array. It will iterate for the length of the array divided by 2. The reason for this is because we are walking the array backwards and forwards to check 2 characters in each loop iteration; therefore, we only need to iterate for half of the array's length. If there are an odd number of characters, the middle index will equal itself--therefore not needing to be checked--and if it's even then the 2 characters we need to check are right next to each other.

Now, you will have 2 indexes in this loop. The first index will go forward, and the second index will go backwards. I will give you an initial hint: utilize the length of the string to get the backwards index: use subtraction to get the backwards index position. Now, when thinking and reasoning about this problem, think of these 2 indexes as "fingers:" draw it on paper if you need to! Kind of like this:

     // in this case the value is 2.

If at any point of the loop iteration the 2 characters at the positions are not equal to each other, return STOP. At the end of the function return CONTINUE: that means the loop has run with no character not matching each other.

In the input_loop function, show a message if the input string is a palindrome or not: return from the input_loop function CONTINUE to continue gathering user input.
